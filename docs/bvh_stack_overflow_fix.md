# BVH 栈溢出问题 - 完整修复报告

## 🐛 问题症状

1. **分界线问题**：画面中出现垂直分界线，左右两侧亮度不同
2. **位置相关**：分界线位置与图片宽度相关
3. **随机性**：每次渲染位置可能不同
4. **黑色像素**：某些像素永远无法正确着色
5. **场景相关**：只在包含1000个变换小球时出现

## 🔍 问题根因

**GPU BVH遍历栈溢出**

- **之前**：`MAX_STACK_SIZE = 64`
- **BVH深度**：16层（3897个节点）
- **问题**：某些光线路径需要 > 64 栈空间
- **后果**：栈溢出时**静默跳过子节点**，导致几何体被错误地culled

## ✅ 完整解决方案

### 1. GPU 端优化

**文件**：`Shaders/Common/Acceleration.metal`

```metal
// 修改前
const int MAX_STACK_SIZE = 64;
if (stack_ptr + 2 <= MAX_STACK_SIZE) {
    stack[stack_ptr++] = second;
    stack[stack_ptr++] = first;
}
// ⚠️ 栈满时直接跳过！

// 修改后
const int MAX_STACK_SIZE = 256;  // 足够应对深度32的BVH
if (stack_ptr + 2 <= MAX_STACK_SIZE) {
    stack[stack_ptr++] = second;
    stack[stack_ptr++] = first;
} else if (stack_ptr + 1 <= MAX_STACK_SIZE) {
    stack[stack_ptr++] = first;  // 至少保留近端节点
}
```

**改进**：
- 栈大小：64 → **256**（4倍）
- 迭代上限：1000 → **10000**
- Fallback策略：优先保留近端节点

### 2. CPU 端优化

**文件**：`Sources/Acceleration/FlatBVH.swift`

```swift
// 新增参数
private let maxLeafSize = 4      // 从2增加到4
private let maxBVHDepth = 24     // 强制深度限制

// 构建时检查
if depth >= maxBVHDepth {
    // 强制创建叶节点，避免树过深
    return BVHNode(bbox: nodeBBox, geometries: leafGeometries)
}
```

**改进**：
- 深度限制：**24层**（足够处理1600万几何体）
- 叶节点容量：2 → **4**（减少节点数）
- 统计输出：显示深度、叶节点数量等信息

## 🔧 技术细节

### 为什么256栈足够？

- BVH遍历采用**近端优先**策略
- 最坏情况：每层2个节点入栈
- 深度24的树：最多 24×2 = 48 栈空间
- 安全系数：256 / 48 = **5.3×**

### 为什么深度24足够？

- 2-way BVH：每层最多分割一半
- 深度24对应：2^24 = **1600万**几何体
- 实际项目：很少超过100万几何体
- 安全系数：16M / 1M = **16×**

### 为什么增大叶节点？

- 减少内部节点数量 → 降低树深度
- 每个叶节点测试4个几何体 vs 遍历2层BVH
- 对于球体和Quad：直接测试更快
